核心功能

快速创建测试

利用 OpenTelemetry 分布式追踪，98% 更快地构建集成和端到端测试。

深度断言能力

不仅验证响应，还能对整个调用链中的任意 span 设置断言。

CI/CD 集成

通过 CLI 在现有流水线中运行测试，失败时附带完整追踪便于根因分析。

生产环境合成监控

将开发/测试阶段编写的测试直接用于生产环境关键路径监控。

工作原理

  1. 触发测试

    调用你的系统接口，TraceTest 自动捕获请求并启动追踪。

  2. 获取完整追踪

    通过轮询算法从你现有的追踪后端（如 Jaeger、Tempo）获取完整 trace。

  3. 执行断言

    基于响应数据和 trace 中的 span 信息，执行你定义的测试断言。

  4. 可视化结果

    在 UI 中清晰展示测试结果、trace 可视化及断言详情。

典型应用场景

微服务集成测试

验证跨多个服务的业务流程是否按预期工作。

可观测性驱动开发

在开发过程中利用 trace 数据确保服务间交互正确。

生产环境关键路径监控

将核心用户旅程转化为持续运行的合成监控测试。
